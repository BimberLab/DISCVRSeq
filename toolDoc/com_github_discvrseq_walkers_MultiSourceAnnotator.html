MultiSourceAnnotator

Transfers annotations from ClinvarAnnotator and Cassandra VCFs to a source VCF

Category Specialized/Internal Tools

Overview

This is a fairly specialized tool, designed to take the VCFs annotated with ClinvarAnnotator (from DISCVR-seq Toolkit) and Cassandra, and transfer select annotations from those VCFs to the input VCF.

Usage example:

  java -jar DISCVRseq.jar MultiSourceAnnotator \
     -cv clinvar.vcf.gz \
     -lr liftoverFailures.vcf \
     -c cassandraVcf.vcf.gz \
     -O output.vcf.gz

Additional Information

Genome/Reference Files

Please note that if this tools uses a reference genome, that FASTA must be indexed with samtools and to have a sequence dictionary created with Picard. See here for more information

Read filters

This Read Filter is automatically applied to the data by the Engine before processing by MultiSourceAnnotator.

  • WellformedReadFilter

--interval-set-rule / -isr

Set merging approach to use for combining interval inputs
By default, the program will take the UNION of all intervals specified using -L and/or -XL. However, you can change this setting for -L, for example if you want to take the INTERSECTION of the sets instead. E.g. to perform the analysis only on chromosome 1 exomes, you could specify -L exomes.intervals -L 1 --interval-set-rule INTERSECTION. However, it is not possible to modify the merging approach for intervals passed using -XL (they will always be merged using UNION). Note that if you specify both -L and -XL, the -XL interval set will be subtracted from the -L interval set.

The --interval-set-rule argument is an enumerated type (IntervalSetRule), which can have one of the following values:

UNION
INTERSECTION
